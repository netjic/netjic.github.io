解压缩常用命令

约 2378 字大约 8 分钟

2025-01-09

gzip命令

  1. 只能压缩单个文件

  2. 默认删除源文件

  3. 压缩率较高

  4. 广泛用于Linux系统

  5. 常与tar命令配合使用

注意事项：

  1. 不能直接压缩目录（gzip -r test/）

    [root@iZuf6hl0gb5y7kuxnpv37zZ ~]# ll
total 12
drwxr-xr-x 4 root root 4096 Jun 24  2024 docker
drwxr-xr-x 2 root root 4096 Jan  9 10:23 test
-rw-r--r-- 1 root root   56 Jan  9 10:10 test.txt.gz
[root@iZuf6hl0gb5y7kuxnpv37zZ ~]# gzip -r test/
[root@iZuf6hl0gb5y7kuxnpv37zZ ~]# cd test/
[root@iZuf6hl0gb5y7kuxnpv37zZ test]# ll
total 8
-rw-r--r-- 1 root root 50 Jan  9 10:20 test1.txt.gz
-rw-r--r-- 1 root root 49 Jan  9 10:19 test.txt.gz
[root@iZuf6hl0gb5y7kuxnpv37zZ test]#

  2. 已经压缩的文件（如jpg、mp3）压缩效果不明显

  3. 默认会删除原文件，需要保留请使用 -k 选项

压缩和解压文件。

gzip命令来自于英文单词gunzip的缩写。gzip是一款使用广泛的压缩工具，文件经过压缩后一般会以.gz后缀结尾，与tar命令合用后即为.tar.gz后缀。

gzip命令对文本文件的压缩比率通常能达到60%~70%，压缩后可以很好的提升存储空间的使用率，还能够在网络传输文件时减少等待时间。

参数说明
-d解开压缩文件
-k保留原文件
-l列出压缩文件的相关信息
-L显示版本与版权信息
-c把压缩后的文件输出到标准输出设备，不去更动原始文件
-r递归处理，将指定目录下的所有文件及子目录一并处理
-v显示指令执行过程
-f强行压缩文件

参考示例：

# 压缩单个文件（会删除原文件）
gzip file.txt          # 生成 file.txt.gz

# 压缩多个文件
gzip file1.txt file2.txt

# 保留原文件压缩
gzip -c file.txt > file.txt.gz
gzip -k file.txt

# 压缩并保留原文件
gzip -kv backup.sql

# 解压单个文件
gzip -d file.txt.gz   

# 批量压缩
gzip -rv /var/log/

# 显示指定文件的压缩信息
gzip -l initial-setup-ks.cfg.gz

gunzip命令

解压提取文件内容。

gunzip命令来自于英文词组”Gnu unzip“的缩写。gunzip通常被用来解压那些被基于gzip格式压缩过的文件，也就是那些.gz结尾的压缩包。

参数说明
-c解压后的文件输出到标准的输出设备
-l列出压缩文件的相关信息
-L显示版本号与版权信息
-r递归处理，将指定目录下的所有文件及子目录一并处理
-v显示指令执行过程
-f强行解开压缩文件

参考示例：

# 解压单个文件
gunzip file.gz        # 会删除.gz文件

# 解压多个文件
gunzip file1.gz file2.gz

# 解压所有.gz文件
gunzip *.gz

# -k 保留原压缩文件
gunzip -k file.gz

# -c 输出到标准输出（不删除原文件）
gunzip -c file.gz > file.txt

# -f 强制解压缩
gunzip -f file.gz

# -l 列出压缩文件信息
gunzip -l file.gz

# -v 显示详细信息
gunzip -v file.gz

# 批量解压
gunzip -v *.gz

# 文件已存在
gunzip -f file.gz    # 使用-f强制覆盖

# 文件损坏
gunzip -t file.gz    # 测试压缩文件完整性

zip命令

压缩文件。

通过zip命令可以将文件打包成.zip格式的压缩包，里面会附含文件的名称、路径、创建时间、上次修改时间等等信息，与tar命令相似。

注意事项：

  1. 与 gzip 不同，zip 可以直接处理目录

  2. 默认不删除源文件

  3. 支持密码保护

  4. 跨平台兼容性好 5. 可以添加或删除压缩包中的文件

参数说明
-d从压缩文件内删除指定的文件
-D压缩文件内不建立目录名称
-r递归处理，将指定目录下的所有文件和子目录一并处理
-u更新已存在的条目，更新已存在的文件，并添加新文件，（和-f的区别）
-d从压缩文件中删除条目
-q安静模式（不显示压缩过程）
-v显示详细信息
-P添加密码 -P password123
-e加密（会提示输入密码）
- 压缩效率压缩效率是一个介于1-9的数值
-f更新已经修改的文件，只更新已存在且已修改的文件（和-u的区别）
-x 范本样式压缩时排除符合条件的文件
-X不保存额外的文件属性
-o以压缩文件内拥有最新更改时间的文件为准，将压缩文件的更改时间设成和该文件相同 更新zip包里的文件
# 压缩单个文件
zip archive.zip file.txt

# 压缩多个文件
zip files.zip file1.txt file2.txt

# 压缩整个目录
zip -r folder.zip directory/

# 压缩当前目录
zip -r archive.zip .

# -q 安静模式（不显示压缩过程）
zip -q quiet.zip file.txt

# -v 显示详细信息
zip -v verbose.zip file.txt

# -P 添加密码
zip -P password123 secure.zip file.txt

# -e 加密（会提示输入密码）
zip -e protected.zip file.txt

# -u 更新已存在的条目
zip -u archive.zip newfile.txt

# -d 从压缩文件中删除条目
zip -d archive.zip file.txt

# -0 到 -9 设置压缩级别
zip -0 store.zip file.txt     # 仅存储，不压缩
zip -9 maximum.zip file.txt   # 最大压缩

# 排除特定文件
zip -r project.zip . -x "*.git/*" "*.DS_Store"

# 压缩并保持文件权限
zip -rX archive.zip directory/

# 压缩后删除原文件
zip -rm archive.zip test/

# 列出压缩文件内容
unzip -l archive.zip

# 测试压缩文件完整性
unzip -t archive.zip

unzip命令

解压缩zip格式文件。

unzip命令用于解压缩zip格式文件，虽然Linux系统中更多的使用tar命令进行对压缩包的管理工作，但有时也会收到同Windows系统常用的.zip和.rar格式的压缩包文件。

参数说明
-c将解压缩的结果显示到屏幕上，并对字符做适当的转换
-f更新现有的文件（更新压缩包解压的同目录名称相同的文件，不是覆盖）
-l显示压缩文件内所包含的文件
-n解压缩时不要覆盖原有的文件
-o不必先询问用户，unzip执行后覆盖原有文件
-q执行时不显示任何信息
-v执行时显示详细的信息
-t检查压缩文件是否正确
-d指定解压到特定目录
# 将压缩包文件解压到当前工作目录中
unzip latest.zip

# 将压缩包文件解压到指定的目录中
unzip latest.zip -d /home

# 测试压缩包文件是否完整，文件有无损坏
unzip -t latest.zip

# 解压到指定目录
unzip archive.zip -d /path/to/directory

# 如果目标目录不存在，会自动创建
unzip archive.zip -d ./new_folder

# 解压到当前目录的子目录
unzip archive.zip -d output_folder

tar命令

压缩和解压缩文件。

tar命令能够制作出Linux系统中常见的.tar、.tar.gz、.tar.bz2等格式的压缩包文件。对于RHEL7、CentOS7版本以后的系统，解压时可以不加压缩格式参数（如z或j），系统能自动进行分析并解压。

参数说明
-c用于创建一个新的 tar 归档文件（覆盖已存在的同名文件）。通常需要和 -f 选项一起使用，指定归档文件名。创建新归档时，源文件不会被删除，"创建新归档"就是创建一个新的打包文件，把指定的文件和目录打包在一起。
-r追加文件到已存在的归档
-u更新已存在归档中的文件
-C压缩指定目录里的内容或解压缩到指定目录，tar -xzf archive.tar.gz -C /target/directory
-x从归档文件中提取文件（解压）
-t列出备份文件的内容（查看tar包里面的文件）。tar -tf archive.tar
-z通过gzip指令压缩/解压缩文件，文件名最好为*.tar.gz
-f指定文件名
-v显示指令执行过程
-j通过bzip2指令压缩/解压缩文件，文件名最好为*.tar.bz2
-p保留原来的文件权限与属性
--remove-files压缩之后删除源文件
-k解压时不覆盖已存在的文件（保留原有文件不覆盖），如果目标目录有同名文件，将保留原有文件
-w确认压缩文件的正确性（解压每个文件前都会询问是否继续）
-P使用文件名的绝对路径，不移除文件名称前的“/”号 (tar -cPf test.tar /root/test/ ,tar -xf test.tar 会在当前目录创建root/test目录)

注意事项：

  1. 选项顺序很重要，-f 必须在最后

  2. 压缩选项（z/j/J）要与文件扩展名对应

  3. 解压时通常不需要指定压缩方式，tar 会自动识别

  4. 大文件压缩时建议使用 -v 查看进度

1. 基本打包（不压缩）

# 创建 tar 包
tar -cf archive.tar file1 file2 directory/

# 解包
tar -xf archive.tar

# 查看内容
tar -tf archive.tar

2. 打包并压缩

# 使用 gzip 压缩（.tar.gz 或 .tgz）
tar -czf archive.tar.gz directory/

# 使用 bzip2 压缩（.tar.bz2）
tar -cjf archive.tar.bz2 directory/

# 使用 xz 压缩（.tar.xz）
tar -cJf archive.tar.xz directory/

3. 解压缩

# 解压 .tar.gz
tar -xzf archive.tar.gz

# 解压 .tar.bz2
tar -xjf archive.tar.bz2

# 解压 .tar.xz
tar -xJf archive.tar.xz

# 解压到指定目录
tar -xzf archive.tar.gz -C /target/directory

4. 常用选项

# -v 显示详细信息
tar -cvzf archive.tar.gz directory/

# -p 保留权限
tar -cpzf backup.tar.gz /etc/

# --exclude 排除文件
tar -czf archive.tar.gz directory/ --exclude="*.log"

# --exclude-from 从文件读取排除列表
tar -czf archive.tar.gz directory/ --exclude-from=exclude.txt

# 排除多个目录
tar -czf archive.tar.gz directory/ \
    --exclude="*/cache" \
    --exclude="*/tmp" \
    --exclude="*.log"
