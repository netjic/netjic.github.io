sed命令

约 750 字大约 3 分钟

2025-05-27

sed (Stream Edito r) 是一个强大的流式文本编辑器，用于对文本进行查找、替换、删除、插入等操作。

默认情况下 sed 不会直接修改原文件，而是将处理后的结果输出到标准输出（屏幕）。如果要直接修改文件，必须使用 -i 选项。

1. 基本语法

sed [选项] '命令' 输入文件
sed [选项] -e '命令1' -e '命令2' 输入文件  # 多个命令
sed [选项] -f 脚本文件 输入文件            # 从文件读取命令

2. 常用选项

  • -n：禁止默认输出，只打印处理的行
  • -e：指定多个编辑命令
  • -f：从脚本文件读取 sed 命令
  • -i：直接修改文件（慎用！）
  • -i：直接修改文件（慎用！）macos sed -i '' 's/ID,/id,/g' alms.sql 。macOS 使用的是 BSD 版本的 sed，要求 -i 后必须跟备份后缀（即使空字符串）
  • -i.bak：修改文件前备份（加 .bak 后缀）
  • -r 或 -E：使用扩展正则表达式

3. 基本命令

替换命令 (s)

sed 's/原字符串/新字符串/' 文件        # 替换每行第一个匹配
sed 's/原字符串/新字符串/g' 文件      # 替换所有匹配（全局）
sed 's/原字符串/新字符串/2' 文件      # 替换每行第2个匹配
sed 's/原字符串/新字符串/ig' 文件     # 忽略大小写并全局替换

删除命令 (d)

sed '3d' 文件            # 删除第3行
sed '1,5d' 文件          # 删除1-5行
sed '/pattern/d' 文件    # 删除匹配的行
sed '/start/,/end/d' 文件 # 删除从"start"到"end"之间的行

start 和 end 的含义:

  • /start/：匹配包含 start 的行（可以是正则表达式）。
  • /end/：匹配包含 end 的行（可以是正则表达式）。
  • /start/,/end/：表示从 第一个匹配 start 的行 到 第一个匹配 end 的行 之间的所有行（包括 start 和 end 所在的行）。
  • d：删除这些行。

打印命令 (p)

sed -n '3p' 文件          # 只打印第3行
sed -n '1,5p' 文件        # 打印1-5行
sed -n '/pattern/p' 文件  # 打印匹配的行

插入/追加命令 (i/a)

sed '3i\插入内容' 文件     # 在第3行前插入
sed '3a\追加内容' 文件     # 在第3行后追加
sed '$a\最后一行追加' 文件  # 在最后一行后追加

替换整行命令 (c)

sed '/pattern/c\新内容' 文件  # 替换匹配的整行

文件修改

sed -i 's/foo/bar/g' 文件          # 直接修改文件（危险！）
# macOs直接修改文件 macOS 使用的是 BSD 版本的 sed，要求 -i 后必须跟备份后缀（即使空字符串）
sed -i '' 's/ID,/id,/g' alms.sql`
sed -i.bak 's/foo/bar/g' 文件      # 修改前创建备份

4. 实用示例

  1. 删除空白行：
sed '/^$/d' 文件
  1. 替换文件扩展名：
sed 's/\.txt$/.log/' 文件
  1. 每行行首添加行号：
sed = 文件 | sed 'N;s/\n/ /'
  1. 提取特定行范围：
sed -n '10,20p' 文件
