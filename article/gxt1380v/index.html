javaOptional使用

约 580 字大约 2 分钟

java

2025-01-08

主要优点：

  • 更清晰地表达可能为空的值
  • 强制开发者处理空值情况
  • 减少 NullPointerException
  • 提供函数式编程风格的API
  • 使代码更加简洁和可读

使用建议：

  • 不要使用 Optional 作为类的字段
  • 不要使用 Optional 作为方法参数
  • 优先使用 Optional 作为返回类型
  • 避免嵌套的 Optional
  • 合理使用 orElse 和 orElseGet
  1. 创建 Optional
// 创建空的 Optional
Optional<String> empty = Optional.empty();

// 创建包含非空值的 Optional
Optional<String> opt = Optional.of("hello");

// 创建可以包含空值的 Optional
Optional<String> nullable = Optional.ofNullable(null);
  1. 检查值是否存在
Optional<String> opt = Optional.of("hello");

// 检查是否有值
boolean exists = opt.isPresent();     

// Java 11+ 检查是否为空
boolean isEmpty = opt.isEmpty();
  1. 获取值
// 如果为空会抛出 NoSuchElementException
String value = opt.get();                     

// 为空时返回默认值
String safe = opt.orElse("default");        

// 为空时通过函数生成默认值
String computed = opt.orElseGet(() -> "计算的默认值");  

// 为空时抛出异常
String throwable = opt.orElseThrow();         

// 为空时抛出自定义异常
String customThrow = opt.orElseThrow(() -> new CustomException());
  1. 条件执行
// 有值时执行
opt.ifPresent(value -> System.out.println(value));      

opt.ifPresentOrElse(
		// 有值时执行
    value -> System.out.println(value),      
    // 无值时执行
    () -> System.out.println("empty")                   
);
  1. 过滤和转换
// filter：过滤值
Optional<String> filtered = opt
    .filter(str -> str.length() > 5);

// map：转换值
Optional<Integer> mapped = opt
    .map(String::length);

// flatMap：处理返回 Optional 的转换
Optional<String> flatMapped = opt
    .flatMap(this::findInDatabase);
  1. 链式操作
String result = Optional.ofNullable(user)
    .filter(u -> u.getAge() > 18)
    .map(User::getEmail)
    .flatMap(this::validateEmail)
    .orElse("Invalid email");
  1. 流式操作
// 转换为 Stream
Stream<String> stream = opt.stream();  // Java 9+

// 在集合中使用
List<Optional<String>> list = Arrays.asList(
    Optional.of("A"),
    Optional.empty(),
    Optional.of("B")
);

List<String> result = list.stream()
    .filter(Optional::isPresent)
    .map(Optional::get)
    .collect(Collectors.toList());
  1. 实际应用示例
// 用户服务示例
public class UserService {
    public Optional<User> findById(Long id) {
        return Optional.ofNullable(userRepository.findById(id));
    }
    
    public String getUserEmail(Long userId) {
        return findById(userId)
            .map(User::getEmail)
            .filter(email -> email.contains("@"))
            .orElseThrow(() -> new UserNotFoundException(userId));
    }
    
    public String getDisplayName(Long userId) {
        return findById(userId)
            .map(User::getName)
            .filter(name -> !name.isEmpty())
            .orElseGet(() -> "Guest" + userId);
    }
}
  1. 注意事项
// 避免这样使用
if (optional.isPresent()) {
    return optional.get();
} else {
    return null;
}
// 应该使用
return optional.orElse(null);

// 避免创建包含 null 的 Optional
Optional.of(null);  // 会抛出 NullPointerException
// 应该使用
Optional.ofNullable(null);

// 注意 orElse 和 orElseGet 的区别
Optional<String> opt = Optional.of("hello");
// expensiveOperation 总是会执行
opt.orElse(expensiveOperation());
// 只在需要时执行
opt.orElseGet(() -> expensiveOperation());
