Autowired和Resource

约 862 字大约 3 分钟

2025-03-11

在 Spring 中，依赖注入（Dependency Injection, DI）可以通过多种方式实现，包括字段注入、构造器注入、Setter 方法注入等。 @Autowired 和 @Resource 是两种常用的注解，用于实现依赖注入。特别是 Setter 方法注入。

1. @Autowired 注解

@Autowired 是 Spring 提供的注解，用于自动装配依赖。它默认按类型（byType）进行注入。

  • 字段（字段注入）
  • 构造函数（构造器注入）
  • Setter 方法（Setter 方法注入）
  • 普通方法（方法注入）
  1. 字段注入：

直接注入到字段上。

@Service
public class UserService {

    @Autowired
    private UserRepository userRepository;
}
  1. 构造器注入 注入到构造函数中（Spring 4.3 以后，如果只有一个构造函数，可以省略 @Autowired）。
@Service
public class UserService {

    private final UserRepository userRepository;

    @Autowired
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}
  1. Setter 方法注入 注入到 Setter 方法中。
@Service
public class UserService {

    private UserRepository userRepository;

    @Autowired
    public void setUserRepository(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}
  1. 方法注入 注入到 Setter 方法中。
@Service
public class UserService {

    private UserRepository userRepository;

    @Autowired
    public void injectDependencies(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}

1.1 特点：

  1. 按类型注入：

默认按类型匹配 Bean。如果找到多个匹配的 Bean，Spring 会抛出 NoUniqueBeanDefinitionException。

@Autowired
private UserRepository userRepository;

如果容器中有多个 UserRepository 类型的 Bean，Spring 会报错。

  1. 解决歧义：

如果存在多个匹配的 Bean，可以使用 @Qualifier 注解指定具体的 Bean 名称。

@Autowired
@Qualifier("userRepositoryImpl")
private UserRepository userRepository;
  1. 可选依赖：

通过设置 required = false，可以将依赖标记为可选的。

@Autowired(required = false)
private UserRepository userRepository;

2. @Resource 注解

@Resource 是 Java 标准注解（javax.annotation.Resource），Spring 也支持它。它默认按名称（byName）进行注入。

  1. 字段注入：

直接注入到字段上。

@Service
public class UserService {

    @Resource
    private UserRepository userRepository;
}
  1. Setter 方法注入：

注入到 Setter 方法中。

@Service
public class UserService {

    private UserRepository userRepository;

    @Resource
    public void setUserRepository(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}

2.1 特点:

  1. 按名称注入：

默认按名称匹配 Bean。如果未指定名称，则使用字段或方法名称。

@Resource
private UserRepository userRepository;

会查找名为 userRepository 的 Bean。

  1. 按类型注入：

如果未找到匹配名称的 Bean，@Resource 会回退到按类型注入。

@Resource
private UserRepository myRepository;

如果容器中没有名为 myRepository 的 Bean，Spring 会查找类型为 UserRepository 的 Bean。 3. 指定名称：

可以通过 name 属性指定 Bean 名称。

@Resource(name = "userRepositoryImpl")
private UserRepository userRepository;

3. @Autowired 和 @Resource 的区别

特性@Autowired@Resource
来源Spring 提供的注解Java 标准注解（javax.annotation）
默认行为按类型注入按名称注入
按名称注入需要结合 @Qualifier 注解直接通过 name 属性指定
是否支持构造函数注入支持不支持
是否支持字段注入支持支持
是否支持 Setter 方法注入支持支持
是否支持方法注入支持不支持
是否支持可选依赖支持（通过 required = false）支持（默认非强制）

4. 特殊注入

ScreenService 接口，多个类实现了这个接口（被 Spring 托管的类），你可以通过注入 Map<String, ScreenService> 来获取所有实现类的实例。

Key：Bean 的名称（默认是类名首字母小写，或者通过 @Component、@Service 等注解指定的名称）。

Value：Bean 的实例。

private final Map<String, ScreenService> screenServiceMap;

也可以编程方式手动获取这些Bean

import org.springframework.context.ApplicationContext;
import org.springframework.stereotype.Service;

import java.util.Map;

@Service
public class ScreenServiceFetcher {

    private final ApplicationContext applicationContext;

    public ScreenServiceFetcher(ApplicationContext applicationContext) {
        this.applicationContext = applicationContext;
    }

    public void fetchAndPrintScreenServices() {
        Map<String, ScreenService> screenServiceMap = applicationContext.getBeansOfType(ScreenService.class);
        screenServiceMap.forEach((name, service) -> 
            System.out.println("Bean Name: " + name + ", Service: " + service)
        );
    }
}
