什么是cglib动态代理

cglib动态代理是基于ASM字节码操作库实现的，它可以在运行时动态生成字节码，从而实现对目标类的代理。

cglib动态代理的优点是：

  • 不需要实现接口
  • 性能较高（相对于JDK动态代理）
  • 可以代理类中的所有非final方法

限制：

  • 不能代理final类
  • 不能代理final方法
  • 不能代理static方法
  • 第一次生成代理类时较慢

首先是目标类：

// 原始类（不需要实现任何接口）
public class UserService {
    public void save(String name, Integer age) {
        System.out.println("保存用户: " + name + ", 年龄: " + age);
    }

    public void update() {
        System.out.println("更新用户");
    }
}

创建拦截器：

import net.sf.cglib.proxy.MethodInterceptor;
import net.sf.cglib.proxy.MethodProxy;

public class LogInterceptor implements MethodInterceptor {
   @Override
    public Object intercept(Object obj, Method method, Object[] args, MethodProxy proxy)
            throws Throwable {

        // 打印方法名和参数
        System.out.println("方法：" + method.getName());
        if (args != null && args.length > 0) {
            System.out.println("参数：" + Arrays.toString(args));
        }

        // 前置处理
        System.out.println("开始执行：" + method.getName());

        // 调用原始方法
        Object result = proxy.invokeSuper(obj, args);

        // 后置处理
        System.out.println("执行结束：" + method.getName());

        return result;
    }
}

创建代理工厂：

import net.sf.cglib.proxy.Enhancer;

public class ProxyFactory {
    public static Object createProxy(Class<?> targetClass) {
        // 创建增强器
        Enhancer enhancer = new Enhancer();
        // 设置父类（目标类）
        enhancer.setSuperclass(targetClass);
        // 设置回调（拦截器）
        enhancer.setCallback(new LogInterceptor());
        // 创建代理对象
        return enhancer.create();
    }
}

使用示例：

public class Main {
    public static void main(String[] args) {
        // 创建代理对象
        UserService userService = (UserService) ProxyFactory.createProxy(UserService.class);

        // 调用方法
        userService.save("张三", 25);  // 正确
        System.out.println("----------------");
        userService.update();
    }
}

CGLib的实现原理：

// CGLib生成的代理类大概是这样的（简化版）
public class UserService$$EnhancerByCGLIB extends UserService {
    private MethodInterceptor interceptor;
    
    @Override
    public void save() {
        try {
            Method method = UserService.class.getDeclaredMethod("save");
            interceptor.intercept(
                this, 
                method, 
                new Object[0], 
                methodProxy
            );
        } catch (Throwable e) {
            throw new RuntimeException(e);
        }
    }
    
    @Override
    public void update() {
        // 类似的代理逻辑
    }
}

常见应用场景：

// 1. 事务管理
public class TransactionInterceptor implements MethodInterceptor {
    @Override
    public Object intercept(...) {
        beginTransaction();
        try {
            Object result = proxy.invokeSuper(obj, args);
            commit();
            return result;
        } catch (Exception e) {
            rollback();
            throw e;
        }
    }
}

// 2. 性能监控
public class PerformanceInterceptor implements MethodInterceptor {
    @Override
    public Object intercept(...) {
        long start = System.currentTimeMillis();
        Object result = proxy.invokeSuper(obj, args);
        long end = System.currentTimeMillis();
        System.out.println("耗时：" + (end - start) + "ms");
        return result;
    }
}

// 3. 权限检查
public class SecurityInterceptor implements MethodInterceptor {
    @Override
    public Object intercept(...) {
        if (!hasPermission()) {
            throw new SecurityException("无权限");
        }
        return proxy.invokeSuper(obj, args);
    }
}
