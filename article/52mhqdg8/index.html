非对称加密原理解释

约 565 字大约 2 分钟

2025-01-13

1. 通俗比喻

1.1 "信箱"模型

小明                                小红
┌─────────────┐                  ┌─────────────┐
│ 公钥（投信口）│                  │ 公钥（投信口）│
│ 私钥（开信钥匙)│                 │ 私钥（开信钥匙)│
└─────────────┘                  └─────────────┘

- 任何人都可以用公钥（投信口）投递信件
- 只有拥有私钥的人才能打开信箱读取信件

1.2 现实生活例子

就像你的邮箱：
- 邮箱地址是公钥（公开的）
- 邮箱密码是私钥（私密的）
- 任何人都可以往你邮箱发邮件
- 只有你知道密码才能读邮件

2. 工作原理

2.1 密钥对的用途

公钥用途：
- 加密数据
- 验证签名

私钥用途：
- 解密数据
- 生成签名

2.2 加密流程

发送方（使用接收方的公钥加密）：
原文 + 对方公钥 = 密文

接收方（使用自己的私钥解密）：
密文 + 自己私钥 = 原文

3. 实际场景

3.1 HTTPS 通信

浏览器                         服务器
获取服务器公钥 ←───────────── 提供公钥证书
用公钥加密数据 ───────────────→ 用私钥解密

3.2 数字签名

发送方：
文件 + 私钥 = 数字签名

接收方：
签名 + 公钥 = 验证文件是否被篡改

4. 优缺点

4.1 优点

- 不需要共享密钥
- 更安全的密钥管理
- 支持数字签名

4.2 缺点

- 计算速度慢
- 加密数据量小
- 资源消耗大

5. 实际应用

5.1 混合加密系统

1. 用非对称加密传输对称密钥
2. 用对称加密传输大量数据

优点：
- 结合两种加密的优势
- 既安全又高效

5.2 代码示例

// 1. 生成密钥对
const { publicKey, privateKey } = generateKeyPair();

// 2. 加密
const encrypted = encrypt(message, publicKey);

// 3. 解密
const decrypted = decrypt(encrypted, privateKey);

6. 生活类比

6.1 银行保管箱

- 银行给你两把钥匙
- 一把可以存钱（公钥）
- 一把可以取钱（私钥）
- 存钱不需要私钥
- 取钱必须用私钥

6.2 酒店房卡

- 前台可以制作房卡（公钥）
- 房卡可以开门（加密）
- 但不能复制房卡（不能解密）

记住：

  • 公钥可以公开
  • 私钥必须保密
  • 加密和解密使用不同的密钥
  • 通常用于密钥交换和数字签名
