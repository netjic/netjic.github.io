1.浏览器的同源策略

1.1 同源策略概述

同源策略是浏览器为确保资源安全，而遵循的一种策略，该策略对访问资源进行了一些限制。

1.2 什么是源（origin）？

源（origin）由协议、域名、端口三部分组成，如果三者都相同，则表示两个源是同源的。

2.3. 限制AJAX获取数据

源A可以给源B发请求，但是无法获取源B响应的数据。

备注：在上述限制中，浏览器对 Ajax 获取数据的限制是影响最大的一个，且实际开发中经常遇到。

项目结构：

3.限制Ajax获取数据
  index.html

    <button onclick="getData()">getData</button>

<script type="text/javascript">
    async function getData() {
        const url = 'https://www.toutiao.com/hot-event/hot-board/?origin=toutiao_pc'
        let result = await fetch(url)
        let data = await result.json()
        console.log(data)
    }

3. 注意点

  1. 跨域限制仅存在浏览器端，服务端不存在跨域限制。
  2. 即使跨域了，Ajax 请求也可以正常发出，但响应数据不会交给开发者。
  3. <link>、<script>、<img>...... 这些标签发出的请求也可能跨域，只不过浏览器对标签跨域不做严格限制，对开发几乎无影响。

4. CORS解决Ajax跨域问题

4.1 CORS概括

CORS 全称：Cross-Origin Resource Sharing（跨域资源共享），是用于控制浏览器校验跨域请求的一套规范，服务器依照 CORS 规范，添加特定响应头来控制浏览器校验，大致规则如下：

  • 服务器明确表示拒绝跨域请求，或没有表示，则浏览器校验不通过。
  • 服务器明确表示允许跨域请求，则浏览器校验通过。

备注说明：使用 CORS 解决跨域是最正统的方式，且要求服务器是“自己人”。

4.2 CORS解决简单请求跨域

整体思路：服务器在给出响应时，通过添加Access-Control-Allow-Origin响应头，来明确表达允许某个源发起跨域请求，随后浏览器在校验时，直接通过。

4.3 简单请求与复杂请求

简单请求复杂请求
请求方法（method）为：GET、HEAD、POST1. 不是简单请求，就是复杂请求。
2. 复杂请求会自动发送预检请求。
请求字段要符合《CORS安全规范》
请求头的Content-Type为：
- application/x-www-form-urlencoded
- multipart/form-data
- text/plain》

关于预检请求：

去询问服务器这个域名或ip，使用请求的方法和携带的请求头是否允许跨域。

  1. 发送时机：预检请求在实际跨域请求之前发出，是由浏览器自动发起的。
  2. 主要作用：用于向服务器确认是否允许接下来的跨域请求。
  3. 基本流程：先发起OPTIONS请求，如果通过预检，继续发起实际的跨域请求。
  4. 请求头内容：一个OPTIONS预检请求，通常会包含如下请求头：
请求头含义
Origin发起跨域请求的源
Access-Control-Request-Method实际跨域请求的Http方法
Access-Control-Request-Headers实际跨域请求的Http头(自定义请求头)

4.4 CORS解决复杂请求跨域

1.第一步：服务器先通过浏览器的预检请求，服务器需要返回如下响应头

响应头含义
Access-Control-Allow-Origin允许跨域请求的源
Access-Control-Allow-Methods允许跨域请求的方法
Access-Control-Allow-Headers允许的自定义头
Access-Control-Max-Age预检请求的结果缓存时间（可选，告诉浏览器多长时间内，不需要再次发送预检请求，可以直接发送实际跨域请求）
  1. 第二步：处理实际的跨域请求（与处理简单请求跨域的方式相同）

项目结构：

crosdemo
  server.js
  2.复杂请求.html

server.js:

const express = require('express')
const app = express()

const students = [
  {
    id: 1,
    name: 'st1'
  },
  {
    id: 2,
    name: 'st2'
  },
  {
    id: 3,
    name: 'st3'
  }
]
app.options('/students', (req, res) => {
  res.setHeader('Access-Control-Allow-Origin', 'http://127.0.0.1:5500')
  res.setHeader('Access-Control-Allow-Method', 'GET')
  res.setHeader('Access-Control-Allow-Headers', 'name,age')
  res.setHeader('Access-Control-Max-Age', 7200)
  res.send()
})
app.get('/students', (req, res) => {
  res.setHeader('Access-Control-Allow-Origin', 'http://127.0.0.1:5500')
  res.send(students)
})

// 启动服务器
app.listen('8081', () => {
  console.log(`Server is running at http://127.0.0.1:8081`)
})

请求头：

app.options('/students', (req, res) => {
  res.setHeader('Access-Control-Allow-Origin', 'http://127.0.0.1:5500')
  res.setHeader('Access-Control-Allow-Method', 'GET')

针对这个options请求，服务器返回的响应头：

  1. Access-Control-Allow-Origin: http://127.0.0.1:5500 允许跨域请求的源
  2. Access-Control-Allow-Method: GET 允许跨域请求的方法
  3. Access-Control-Allow-Headers: name,age 允许的自定义头
  4. Access-Control-Max-Age: 7200 预检请求的结果缓存时间

4.5 借助cors 库快速完成配置（了解）

上面的请求需要每个接口都要写个Options请求，比较麻烦，可以使用cors库快速完成配置

  • 安装cors
npm i cors
  • 简单配置cors
app.use(cors())
  • 完整配置cors
// cors中间件配置
const corsOptions = {
  origin: 'http://127.0.0.1:5500', // 允许的源
  methods: ['GET', 'POST', 'PUT', 'DELETE', 'HEAD', 'OPTIONS'], // 允许的方法
  allowedHeaders: ['school'], // 允许的自定义头
  exposedHeaders: ['abc'], // 要暴露的响应头
  optionsSuccessStatus: 200 // 预检请求成功的状态码
};

app.use(cors(corsOptions)); // 使用cors中间件

5. JSONP解决跨域问题(了解)

  1. JSONP 概述： JSONP 是利用了<script>标签可以跨域加载脚本，且不受严格限制的特性，可以说是程序员智慧的结晶，早期一些浏览器不支持 CORS 的时，可以靠 JSONP 解决跨域。
  2. 基本流程：
  • 第一步：客户端创建一个<script>标签，并将其src属性设置为包含跨域请求的 URL，同时准备一个回调函数，这个回调函数用于处理返回的数据。
  • 第二步：服务端接收到请求后，将数据封装在回调函数中并返回。
  • 第三步：客户端的回调函数被调用，数据以参数的形势传入回调函数。
  1. 代码示例：

使用标签请求get请求

<script src="http://127.0.0.1:8081/teachers" />
<button onclick="getTeachers()">获取数据</button>

<script type="text/javascript" >
  function callback(data){
    console.log(data)
  }

  function getTeachers(url){
    // 创建script元素
    const script = document.createElement('script')
    // 指定script的src属性
    script.src= 'http://127.0.0.1:8081/teachers'
    // 将script元素添加到body中触发脚本加载
    document.body.appendChild(script)
    // script标签加载完毕后移除该标签
    script.onload = ()=>{
      script.remove()
    }
  }
</script>

服务器返回字符串 callback({id:1,name:'lisi'})

就相当于：

function callback(data){
    console.log(data)
}
// 服务器返回的数据
callback({id:1,name:'lisi'})
// 调用了本地的方法€€

6. 配置代理解决跨域啊

例如：

vue项目中，脚手架启动了一个本地的服务，页面发送Ajax请求，uri可以统一加个/api前缀，请求 localhost:8080/api/students 本地服务， 前端页面和服务器是同源的，本地服务器接收到请求时，可以根据配置去掉/api，然后直接转发到目标服务器 126.0.0.1:8081/students， 目标服务器返回结果，本地服务器接收到结果后，返回给前端页面。这个利用的是服务器之间请求不存在跨域问题。
