约 1335 字大约 4 分钟

Bean的作用域

  1. Singleton (单例): 这是默认的作用域。当一个Bean被定义为singleton时，Spring容器在整个应用上下文中只会创建该Bean的一个实例。这个单一实例会被所有对该Bean的请求共享。 使用场景：大多数情况下使用，特别是对于无状态的Bean。
  2. Prototype (原型): 每次对这种作用域的Bean发起请求时，都会创建一个新的实例。这对于需要每次请求都获得不同状态的Bean非常有用。 使用场景：适用于有状态的对象，比如复杂的业务对象或数据传输对象(DTO)。
  3. Request (请求): 在一次HTTP请求中，该作用域的Bean是单例的。每个HTTP请求都会创建一个新的Bean实例，并且该实例只在这个请求范围内有效。 使用场景：Web应用中处理用户请求相关的Bean。
  4. Session (会话): 在一次HTTP会话中，该作用域的Bean是单例的。每个HTTP会话都会创建一个新的Bean实例，并且该实例只在这个会话范围内有效。 使用场景：Web应用中保存用户特定信息的场景。
  5. Application (应用): 该作用域下的Bean与ServletContext绑定，在整个Web应用范围内都是单例的。 使用场景：适合那些需要在整个Web应用生命周期内保持一致性的场景。
  6. WebSocket: 该作用域下的Bean与一个WebSocket会话绑定，在整个WebSocket会话期间内有效。

指定Bean的作用域

  1. 使用注解: 对于基于注解的配置，可以使用@Scope注解来指定作用域。例如，要定义一个原型作用域的Bean，可以这样做：
import org.springframework.context.annotation.Scope;
import org.springframework.stereotype.Component;

@Component
@Scope("prototype")
public class MyPrototypeBean {
    // Bean implementation
}
  1. XML配置: 如果你在使用XML配置Spring，则可以通过<bean>元素的scope属性来指定作用域。
<bean id="myPrototypeBean" class="com.example.MyPrototypeBean" scope="prototype"/>

获取原型（Prototype）作用域的Bean

  1. 通过 ApplicationContext 获取 可以通过注入的ApplicationContext来手动获取原型作用域的Bean。这种方法适用于那些需要动态地、根据运行时条件创建新实例的场景。
@Autowired
private ApplicationContext applicationContext;

public void handleFeedback() {
    FeedbackForm feedbackForm = applicationContext.getBean(FeedbackForm.class);
    // 使用 feedbackForm 进行操作
}
  1. 通过依赖注入获取 虽然直接依赖注入对于单例作用域的Bean非常常见，但也可以用于原型作用域的Bean。然而，需要注意的是，如果直接将原型作用域的Bean注入到另一个长生命周期的Bean（如单例），那么这个“原型”Bean实际上只会被创建一次，并且在整个容器生命周期内共享使用。因此，这种情况下并不是每次请求都得到一个新的实例。 为了确保每次都能获得新的实例，可以使用ObjectFactory或Provider接口。

使用 ObjectFactory

@Component
public class FeedbackHandler {

    @Autowired
    private ObjectFactory<FeedbackForm> feedbackFormFactory;

    public void handleFeedback() {
        FeedbackForm feedbackForm = feedbackFormFactory.getObject();
        // 每次调用 getObject() 都会返回一个新的 FeedbackForm 实例
    }
}

使用 avax.inject.Provider 如果你更倾向于使用Java标准的依赖注入规范，可以使用javax.inject.Provider接口。

@Component
public class FeedbackHandler {

    @Autowired
    private Provider<FeedbackForm> feedbackFormProvider;

    public void handleFeedback() {
        FeedbackForm feedbackForm = feedbackFormProvider.get();
        // 每次调用 get() 都会返回一个新的 FeedbackForm 实例
    }
}

使用 通过 @Lookup 注解（方法注入）

你可以定义一个抽象方法，并使用@Lookup注解让Spring为该方法生成具体的实现，这样每次调用该方法都会返回一个新的原型作用域Bean实例。

@Component
public abstract class FeedbackHandler {

    @Lookup
    protected abstract FeedbackForm createFeedbackForm();

    public void handleFeedback() {
        FeedbackForm feedbackForm = createFeedbackForm();
        // 每次调用 createFeedbackForm() 都会返回一个新的 FeedbackForm 实例
    }
}
  • ApplicationContext.getBean()：适合于需要动态地、根据运行时条件获取Bean实例的场景。
  • ObjectFactory 或 Provider：这两种方式更适合在构造函数或setter注入无法满足需求时使用，特别是在你需要确保每次获取都是新实例的情况下。
  • @Lookup 方法注入：提供了一种声明式的方式来获取原型作用域的Bean，特别适合那些不想暴露ApplicationContext但仍需按需创建新实例的场景。

请求（Request）作用域

  1. 定义 Request 作用域 Bean
import org.springframework.context.annotation.Scope;
import org.springframework.stereotype.Component;

@Component
@Scope("request")
public class RequestScopedBean {
    public void doSomething() {
        System.out.println("执行请求范围内的操作");
    }
}
  1. 获取 Request 作用域 Bean 对于请求作用域的Bean，可以通过依赖注入的方式直接获取，因为Spring会在每个HTTP请求开始时创建一个新的实例，并在请求结束时销毁它。
@Controller
public class MyController {

    @Autowired
    private RequestScopedBean requestScopedBean;

    @GetMapping("/doSomething")
    public String handleRequest() {
        requestScopedBean.doSomething();
        return "success";
    }
}

会话（Session）作用域

  1. 定义 Session 作用域 Bean
import org.springframework.context.annotation.Scope;
import org.springframework.stereotype.Component;

@Component
@Scope("session")
public class SessionScopedBean {
    public void doSomething() {
        System.out.println("执行会话范围内的操作");
    }
}
  1. 获取 Session 作用域 Bean

与请求作用域类似，可以通过依赖注入来获取会话作用域的Bean。Spring会在用户首次访问应用时为该用户的会话创建一个实例，并在整个会话期间保持这个实例。

@Controller
public class MyController {

    @Autowired
    private SessionScopedBean sessionScopedBean;

    @GetMapping("/doSomething")
    public String handleRequest() {
        sessionScopedBean.doSomething();
        return "success";
    }
}
