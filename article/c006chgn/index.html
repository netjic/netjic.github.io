1. 基本配置参数

 @Bean("doSomethingExecutor")
 public ThreadPoolTaskExecutor threadPoolTaskExecutor() {
     ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();

     // 1. 核心线程数
     executor.setCorePoolSize(10);

     // 2. 最大线程数
     executor.setMaxPoolSize(20);

     // 3. 队列容量
     executor.setQueueCapacity(100);

     // 4. 线程名前缀
     executor.setThreadNamePrefix("custom-executor-");

     // 5. 空闲线程存活时间
     executor.setKeepAliveSeconds(60);

     // 6. 拒绝策略
     executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());

     // 7. 初始化
     executor.initialize();

     return executor;
 }

2. 参数详细说明

2.1.核心参数：

2.1.1.核心线程数：setCorePoolSize()

// 核心线程数：线程池创建时初始化的线程数
executor.setCorePoolSize(10);

2.1.2.最大线程数：setMaxPoolSize(20)

// 最大线程数：当队列满了后最多能创建的线程数
executor.setMaxPoolSize(20);

2.1.3.队列容量：setQueueCapacity(20)


// 队列容量：等待执行的任务队列容量
executor.setQueueCapacity(100);

示例：

  1. 不超过核心线程数的情况
ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
executor.setCorePoolSize(10);    // 核心线程数10
executor.setMaxPoolSize(20);     // 最大线程数20
executor.setQueueCapacity(100);  // 队列容量100

// 场景：提交8个任务
for (int i = 0; i < 8; i++) {
    executor.execute(() -> {
        log.info("任务执行，线程：{}", Thread.currentThread().getName());
    });
}

处理流程：

提交8个任务：
- 直接创建8个核心线程
- 每个任务都由一个新的核心线程处理
- 不会使用队列
- 不会创建非核心线程
  1. 任务超过核心线程池
ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
executor.setCorePoolSize(10);    // 核心线程数10
executor.setMaxPoolSize(20);     // 最大线程数20
executor.setQueueCapacity(100);  // 队列容量100

// 场景：提交50个任务
for (int i = 0; i < 50; i++) {
    final int taskId = i;
    executor.execute(() -> {
        log.info("任务-{} 执行，线程：{}", 
                taskId, 
                Thread.currentThread().getName());
    });
}

处理流程：

提交50个任务：
1. 前10个任务：
   - 创建10个核心线程处理

2. 后40个任务：
   - 进入队列等待
   - 不会创建新线程
   - 由核心线程依次处理队列中的任务
  1. 任务超过核心线程池
ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
executor.setCorePoolSize(10);    // 核心线程数10
executor.setMaxPoolSize(20);     // 最大线程数20
executor.setQueueCapacity(100);  // 队列容量100

// 场景：提交150个任务
for (int i = 0; i < 150; i++) {
    final int taskId = i;
    executor.execute(() -> {
        log.info("任务-{} 执行，线程：{}", 
                taskId, 
                Thread.currentThread().getName());
    });
}

处理流程：

提交150个任务：
1. 前10个任务：
   - 创建10个核心线程处理

2. 接下来100个任务：
   - 进入队列等待

3. 接下来10个任务：
   - 创建10个非核心线程处理（达到最大线程数20）

4. 最后30个任务：
   - 触发拒绝策略

关键点总结

  1. 场景1（轻负载）：

    • 只使用核心线程
    • 不使用队列
    • 不创建额外线程

  2. 场景2（中负载）：

    • 核心线程全部使用
    • 使用队列缓冲
    • 不创建额外线程

  3. 场景3（重负载）：

    • 核心线程全部使用
    • 队列完全填满
    • 创建额外线程到最大值
    • 触发拒绝策略

这就是线程池的三种典型负载场景及其处理方式。理解这些场景有助于我们更好地配置线程池参数。

2.2.线程管理参数：

2.2.1.线程名前缀 setThreadNamePrefix()：

线程名前缀：方便定位问题,打印日志可以看到那个线程执行的。

// 线程名前缀：方便定位问题,打印日志可以看到那个线程执行的
executor.setThreadNamePrefix("custom-executor-");
// 2025-01-09T19:56:17.470+08:00  INFO 81639 --- [springBootThread] [ do-something-4] c.n.s.service.AsyncService               : do something, message=index = 19

2.2.2.空闲线程存活时间 setKeepAliveSeconds()：

空闲时会计时，超过keepAliveSeconds时间，就会被回收（释放）

// 空闲线程存活时间（秒）
executor.setKeepAliveSeconds(60);

2.2.3.是否允许核心线程超时 setAllowCoreThreadTimeOut()：

  1. 设置为true时。

所有线程（包括核心线程）：空闲时会计时，超过keepAliveSeconds时间，就会被回收（释放）。

  1. 设置为false时。

核心线程：即使空闲也会一直存活，永远不会被回收，不考虑keepAliveSeconds参数

非核心线程：空闲超过keepAliveSeconds时会被回收

// 是否允许核心线程超时
executor.setAllowCoreThreadTimeOut(true);

注意事项：

  1. 性能考虑：

    • 开启核心线程超时可能导致频繁创建和销毁线程
    • 如果负载稳定，建议保持默认设置（false）

  2. 资源利用：

    • 开启超时可以更好地利用系统资源
    • 适合负载不稳定的场景

  3. 响应时间：

    • 开启超时可能导致首次任务执行延迟（需要创建新线程）
    • 对响应时间敏感的系统要谨慎使用

  4. 配置建议：

    // 负载稳定的系统
executor.setAllowCoreThreadTimeOut(false);
executor.setKeepAliveSeconds(60);

// 负载波动大的系统
executor.setAllowCoreThreadTimeOut(true);
executor.setKeepAliveSeconds(300);  // 设置更长的存活时间

    这个参数的使用需要根据具体的业务场景来决定，没有统一的最佳实践，关键是要理解它的作用和影响。

2.3.任务拒绝策略：

  1. AbortPolicy：抛出异常（默认）
// AbortPolicy：抛出异常（默认）
executor.setRejectedExecutionHandler(new ThreadPoolExecutor.AbortPolicy());

  1. CallerRunsPolicy：调用者线程执行

    使用的是调用这个方法的线程。

    2025-01-09T20:11:54.241+08:00  INFO 83040 --- [springBootThread] [do-something-19] c.n.s.service.AsyncService               : do something, message=index = 118
2025-01-09T20:11:54.241+08:00  INFO 83040 --- [springBootThread] [nio-8081-exec-1] c.n.s.service.AsyncService               : do something, message=index = 120
// CallerRunsPolicy：调用者线程执行
executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
  1. DiscardPolicy：直接丢弃
// DiscardPolicy：直接丢弃
executor.setRejectedExecutionHandler(new ThreadPoolExecutor.DiscardPolicy());
  1. DiscardOldestPolicy：丢弃最旧的任务
// DiscardOldestPolicy：丢弃最旧的任务
executor.setRejectedExecutionHandler(new ThreadPoolExecutor.DiscardOldestPolicy());

3.高级配置

@Configuration
public class AdvancedThreadPoolConfig {
    
    @Bean("advancedExecutor")
    public ThreadPoolTaskExecutor threadPoolTaskExecutor() {
        ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
        
        // 基本配置
        executor.setCorePoolSize(Runtime.getRuntime().availableProcessors());
        executor.setMaxPoolSize(Runtime.getRuntime().availableProcessors() * 2);
        executor.setQueueCapacity(500);
        executor.setThreadNamePrefix("advanced-executor-");
        
        // 等待任务完成再关闭线程池
        executor.setWaitForTasksToCompleteOnShutdown(true);
        executor.setAwaitTerminationSeconds(60);
        
        // 自定义拒绝策略
        executor.setRejectedExecutionHandler((r, e) -> {
            log.warn("任务被拒绝执行");
            // 可以添加监控或告警逻辑
        });
        
        // 初始化
        executor.initialize();
        
        return executor;
    }
}

4.使用示例

springboot也可以直接使用@Asysnc注解

    // 指定线程池
    @Async("doSomethingExecutor")
    public void doSomething(String message) {
        log.info("do something, message={}", message);
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            log.error("do something error: ", e);
        }
    }
@Service
@Slf4j
public class AsyncService {
    
    private final ThreadPoolTaskExecutor executor;
    
    public AsyncService(@Qualifier("customExecutor") ThreadPoolTaskExecutor executor) {
        this.executor = executor;
    }
    
    public void executeTask(String taskName) {
        executor.execute(() -> {
            log.info("执行任务: {}, 线程: {}", 
                    taskName, 
                    Thread.currentThread().getName());
            // 任务逻辑
        });
    }
    
    public Future<String> submitTask(String taskName) {
        return executor.submit(() -> {
            log.info("提交任务: {}, 线程: {}", 
                    taskName, 
                    Thread.currentThread().getName());
            return "Task " + taskName + " completed";
        });
    }
}

5. 监控示例

代码：

@Configuration
@EnableAsync
@Slf4j
@EnableScheduling  // 开启定时任务支持
public class AsyncConfiguration {}
    @Scheduled(fixedRate = 5000)
    public void monitorThreadPool() {
        ThreadPoolExecutor threadPool = executor.getThreadPoolExecutor();

        log.info("=== 线程池状态 ===");
        log.info("活跃线程数: {}", threadPool.getActiveCount());
        log.info("核心线程数: {}", threadPool.getCorePoolSize());
        log.info("最大线程数: {}", threadPool.getMaximumPoolSize());
        log.info("线程池大小: {}", threadPool.getPoolSize());
        log.info("队列任务数: {}", threadPool.getQueue().size());
        log.info("总完成任务数: {}", threadPool.getCompletedTaskCount());
        log.info("================");
    }
