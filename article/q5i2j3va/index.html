Runnable接口类型参数

约 367 字大约 1 分钟

2025-03-19

Runnable 是 Java 中的一个接口，它只有一个抽象方法 run()，这意味着任何实现了 run() 方法的对象都可以作为 Runnable 传递给 tryLock 方法。

@Override
public boolean tryLock(String key, Runnable run) {
    // 获取锁
    RLock lock = redissonClient.getLock(key);
    // 未获取到直接返回
    if (!lock.tryLock()) {
        log.info("RedisLocker.tryLock failed {}", key);
        return false;
    }
    // 执行
    try {
        run.run();
    } finally {
        lock.unlock();
    }
    return true;
}

  1. 匿名内部类：这是使用 Runnable 的一种传统方法。

    locker.tryLock("someKey", new Runnable() {
    @Override
    public void run() {
        // 需要加锁执行的代码
    }
});

  2. Lambda 表达式：如果你使用的是 Java 8 或更高版本，可以利用 Lambda 表达式简化代码。

    locker.tryLock("someKey", () -> {
    // 需要加锁执行的代码
});

    或者，如果函数体只有一行代码：

    locker.tryLock("someKey", () -> System.out.println("需要加锁执行的操作"));

  3. 方法引用：如果你有一个现有的方法符合 Runnable 接口的签名（即无参数且返回类型为 void），你可以直接引用这个方法。

    locker.tryLock("someKey", YourClass::yourMethod);

    其中 yourMethod 定义如下：

    public void yourMethod() {
    // 需要加锁执行的代码
}

  4. 实现 Runnable 接口的类实例：你可以创建一个单独的类来实现 Runnable 接口，并重写 run() 方法。然后，你可以创建该类的实例并将其传递给 tryLock 方法。

    java深色版本

    public class MyTask implements Runnable {
    @Override
    public void run() {
        // 需要加锁执行的代码
    }
}

locker.tryLock("someKey", new MyTask());
