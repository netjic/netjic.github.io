1. 配置文件可以分为

  • /etc/profile
  • /etc/profile.d/*.sh
  • /etc/bashrc
  • /home/用户名/.bash_profile
  • /home/用户名/.bashrc

2.系统级登录配置

通过查看/etc/profile cat /etc/profile

for i in /etc/profile.d/*.sh /etc/profile.d/sh.local ; do
    if [ -r "$i" ]; then
        if [ "${-#*i}" != "$-" ]; then 
            . "$i"
        else
            . "$i" >/dev/null
        fi
    fi
done

unset i
unset -f pathmunge

if [ -n "${BASH_VERSION-}" ] ; then
        if [ -f /etc/bashrc ] ; then
                # Bash login shells run only /etc/profile
                # Bash non-login shells run only /etc/bashrc
                # Check for double sourcing is done in /etc/bashrc.
                . /etc/bashrc
       fi
fi

执行/etc/profile时，依次调用的 /etc/profile.d/*.sh/，/etc/bashrc

执行时机

# 登录 shell（Login Shell）时执行：
- 用户通过终端登录系统时
- 通过 ssh 远程登录时
- 使用 su - username 切换用户时
- 使用 sudo -i 获取 root 权限时

执行时机验证方法：

# 在 /etc/profile 末尾添加
echo "Loading /etc/profile..."

3. 用户级启动文件

~/.bash_profile  # 用户登录时执行
~/.bashrc        # 用户打开终端时执行

4.使用场景

  1. /etc/profile（系统级配置）：
# 适合放置：系统级环境变量
export JAVA_HOME=/usr/local/jdk
export PATH=$PATH:$JAVA_HOME/bin
  1. ~/.bash_profile（用户级登录配置）：
# 适合放置：用户特定的环境变量
export MY_PROJECT_DIR=$HOME/projects
export MAVEN_OPTS="-Xmx2048m"
# 个人专用path
  1. ~/.bashrc（用户级交互配置）：
# 适合放置：
- 命令别名
alias ll='ls -l'
alias gs='git status'

- 函数定义
mkcd() {
    mkdir -p "$1" && cd "$1"
}
