Springboot构造器注入

约 1180 字大约 4 分钟

2025-03-11

在 Spring Boot 中，构造器注入（Constructor Injection） 是一种推荐的依赖注入方式。 它通过构造函数将依赖注入到类中，而不是通过字段或 setter 方法。这种方式具有以下优点：

  1. 不可变性：依赖字段可以声明为 final，确保它们在对象创建后不会被修改。
  2. 明确性：构造函数清晰地表明了类的依赖关系，便于理解和测试。
  3. 安全性：避免了循环依赖问题，Spring 会在启动时检测到循环依赖并抛出异常。
  4. 兼容性：与单元测试框架（如 JUnit）更好地集成，因为可以直接通过构造函数注入依赖。

2. 在 Spring Boot 中，构造器注入可以通过以下方式实现：

2.1. 手动编写构造函数

你可以手动编写一个构造函数，Spring 会自动调用它并注入所需的依赖。

import org.springframework.stereotype.Service;

@Service
public class UserService {

    private final UserRepository userRepository;

    // 构造函数注入
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    public void doSomething() {
        userRepository.findUser();
    }
}

2.2. 使用 Lombok 的 @RequiredArgsConstructor

如果使用 Lombok 的 @RequiredArgsConstructor，它会自动生成一个包含所有 final 字段的构造函数。

如果有多个构造函数，Lombok 会报错，因为它无法确定应该生成哪个构造函数。

import lombok.RequiredArgsConstructor;
import org.springframework.stereotype.Service;

@Service
@RequiredArgsConstructor
public class UserService {

    private final UserRepository userRepository;

    public void doSomething() {
        userRepository.findUser();
    }
}

编译后，Lombok 会生成以下代码：

import org.springframework.stereotype.Service;

@Service
public class UserService {

    private final UserRepository userRepository;

    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }

    public void doSomething() {
        userRepository.findUser();
    }
}

2.3. 多个有参构造

  1. 使用 @Autowired 注解：
    • 在需要使用的构造函数上添加 @Autowired 注解，明确告诉 Spring 使用该构造函数。
    @Service
public class SomeService {

   private final UserService userService;

   @Autowired // 明确指定使用这个构造函数
   public SomeService(UserService userService) {
       this.userService = userService;
   }

   public SomeService(UserService userService, AnotherService anotherService) {
       this.userService = userService;
   }
}
  2. 只有一个构造函数时省略 @Autowired：
    • 从 Spring 4.3 开始，如果类中只有一个构造函数，可以省略 @Autowired 注解，Spring 会自动使用该构造函数。
    @Service
public class SomeService {

    private final UserService userService;

    public SomeService(UserService userService) {
        this.userService = userService;
    }
}

3. 构造器注入的工作原理

  1. Spring 容器启动时：
    • Spring 会扫描所有带有 @Component、@Service、@Repository 等注解的类。
    • 如果发现某个类有构造函数，Spring 会尝试通过构造函数注入所需的依赖。
  2. 依赖解析：
    • Spring 会根据构造函数的参数类型，从容器中查找对应的 Bean。
    • 如果找到匹配的 Bean，则注入；如果没有找到或存在多个匹配的 Bean，则会抛出异常。
  3. Bean 创建：
    • 当所有依赖都成功注入后，Spring 会创建该类的实例并将其纳入容器管理。

4. 构造器注入的优点

  1. 不可变性：依赖字段可以声明为 final，确保它们在对象创建后不会被修改。
private final UserRepository userRepository;
  1. 明确性：构造函数清晰地表明了类的依赖关系，便于理解和测试。
public UserService(UserRepository userRepository) {
    this.userRepository = userRepository;
}
  1. 安全性：避免了循环依赖问题。如果存在循环依赖，Spring 会在启动时抛出异常。
// UserService 依赖 OrderService
public UserService(OrderService orderService) { ... }

// OrderService 依赖 UserService
public OrderService(UserService userService) { ... }
  1. 兼容性： 与单元测试框架更好地集成，因为可以直接通过构造函数注入依赖。

5. 构造器注入的注意事项

  1. 避免多个构造函数：
    • 如果一个类有多个构造函数，Spring 需要明确知道使用哪个构造函数进行注入。
    • 可以通过 @Autowired 注解指定使用的构造函数（Spring 4.3 以后，如果只有一个构造函数，可以省略 @Autowired）。
@Service
public class UserService {

    private final UserRepository userRepository;

    @Autowired // 显式指定使用这个构造函数
    public UserService(UserRepository userRepository) {
        this.userRepository = userRepository;
    }
}

  1. 避免循环依赖：

    • 如果两个类相互依赖（例如 UserService 依赖 OrderService，而 OrderService 又依赖 UserService），Spring 会抛出 BeanCurrentlyInCreationException。
    • 解决方法：重新设计代码，避免循环依赖，或者使用 @Lazy 延迟加载。

  2. 确保依赖的 Bean 存在：

    • 如果构造函数中需要的 Bean 不存在，Spring 会抛出 NoSuchBeanDefinitionException。
    • 确保所有依赖的 Bean 都被正确注册到 Spring 容器中。

6. 总结：

  • 构造器注入 是 Spring Boot 中推荐的依赖注入方式，具有不可变性、明确性和安全性等优点。
  • 可以通过手动编写构造函数或使用 Lombok 的 @RequiredArgsConstructor 来实现。
  • 避免多个构造函数和循环依赖问题，确保依赖的 Bean 存在。
  • 相比于字段注入和 Setter 注入，构造器注入更适合现代 Spring 应用的开发。
