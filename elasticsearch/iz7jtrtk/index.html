1. 索引

在 Elasticsearch 中，索引（Index）类似于关系数据库中的表（Table）。

  • 索引和表都是存储数据的逻辑容器。
  • 索引中的文档（Document）类似于表中的行（Row）。
  • 索引中的字段（Field）类似于表中的列（Column）。

映射（Mapping）用于定义索引中的字段类型，类似于关系数据库表结构。

创建索引时并不强制要求定义 Mapping（映射关系）。Elasticsearch 具有动态映射（Dynamic Mapping）的功能， 这意味着如果你没有显式定义 Mapping，Elasticsearch 会根据你插入的文档自动推断字段类型并创建 Mapping。

1.2.2. 核心字段类型

(1) 文本类型（Text）
  • 类型名：text
  • 适用场景：
    • 用于全文搜索的字段，例如文章内容、描述、评论等。
    • 支持分词（Analyzed），可以将文本拆分为单词或词组。
  • 示例：
    {
  "mappings": {
    "properties": {
      "content": {
        "type": "text"
      }
    }
  }
}
  • 注意事项：text
    • text 类型的字段默认会生成一个 keyword 子字段，用于精确匹配。
关键字类型（Keyword）
  • 类型名：keyword
  • 适用场景：
    • 用于精确匹配的字段，例如 ID、状态标签、分类等。
    • 不支持分词，整个字段值作为一个整体进行索引。
    • 适合排序、聚合和精确查询。
  • 示例：

  {
  "mappings": {
    "properties": {
      "status": {
        "type": "keyword"
      }
    }
  }
}
  • 注意事项：text
    • 如果字段值较长，可以使用 ignore_above 参数限制索引的长度。
数值类型（Numeric）

Elasticsearch 支持多种数值类型，适用于不同的数值范围：

  • 整数类型：
    • byte：8 位整数。
    • short：16 位整数。
    • integer：32 位整数。
    • long：64 位整数。
  • 浮点数类型：
    • float：32 位整数。
    • double：64 位浮点数。
  • 适用场景：
    • 用于存储数值数据，例如年龄、价格、评分等。
    • 适合范围查询、排序和聚合。
  • 示例：

{
  "mappings": {
    "properties": {
      "age": {
        "type": "integer"
      },
      "price": {
        "type": "float"
      }
    }
  }
}
日期类型（Date）
  • 类型名：date
  • 适用场景：
    • 用于存储日期和时间，例如创建时间、更新时间等。
    • 支持日期格式化和范围查询。
  • 示例：

{
  "mappings": {
    "properties": {
      "created_at": {
        "type": "date",
        "format": "yyyy-MM-dd HH:mm:ss||epoch_millis"
      }
    }
  }
}
  • 注意事项：text
    • 可以指定日期格式（format），支持多种格式。
布尔类型（Boolean）
  • 类型名：boolean
  • 适用场景：
    • 用于存储布尔值（true 或 false），例如是否启用、是否完成等。
  • 示例：

{
  "mappings": {
    "properties": {
      "is_active": {
        "type": "boolean"
      }
    }
  }
}
二进制类型（Binary）
  • 类型名：binary
  • 适用场景：
    • 用于存储二进制数据，例如图片、文件等。
  • 示例：
{
  "mappings": {
    "properties": {
      "attachment": {
        "type": "binary"
      }
    }
  }
}
范围类型（Range）
  • 类型名：integer_range，float_range，date_range
  • 适用场景：
    • 用于存储范围值，例如时间段、价格区间等。
  • 示例：
{
  "mappings": {
    "properties": {
      "price_range": {
        "type": "float_range"
      }
    }
  }
}

1.2.3. 复杂字段类型

(1) 对象类型（Object）
  • 类型名：object
  • 适用场景：
    • 用于存储 JSON 对象，例如用户信息中的地址。
  • 示例：
  {
  "mappings": {
    "properties": {
      "address": {
        "type": "object",
        "properties": {
          "street": { "type": "text" },
          "city": { "type": "keyword" }
        }
      }
    }
  }
}
(2) 嵌套类型（Nested）
  • 类型名：nested
  • 适用场景：
    • 用于存储数组中的对象，且需要独立查询每个对象。
    • 与 object 类型不同，nested 类型会为数组中的每个对象创建独立的文档。
  • 示例：
  {
  "mappings": {
    "properties": {
      "comments": {
        "type": "nested",
        "properties": {
          "author": { "type": "keyword" },
          "content": { "type": "text" }
        }
      }
    }
  }
}
(3) 数组类型（Array）
  • 类型名：由数组元素的类型决定。
  • 适用场景：
    • 用于存储多个值的字段，例如标签、分类等。
  • 示例：
  {
  "mappings": {
    "properties": {
      "tags": {
        "type": "keyword"
      }
    }
  }
}

1.2.4. 地理字段类型

(1) 地理点类型（Geo-point）
  • 类型名：geo_point。
  • 适用场景：
    • 用于存储经纬度坐标，例如地理位置。
  • 示例：
{
  "mappings": {
    "properties": {
      "location": {
        "type": "geo_point"
      }
    }
  }
}
(2) 地理形状类型（Geo-shape）
  • 类型名：geo_shape。
  • 适用场景：
    • 用于存储复杂的地理形状，例如多边形、线等。
  • 示例：
{
  "mappings": {
    "properties": {
      "shape": {
        "type": "geo_shape"
      }
    }
  }
}

1.2.5. 特殊字段类型

(1) IP 类型（IP）
  • 类型名：ip。
  • 适用场景：
    • 用于存储 IP 地址。
  • 示例：
{
  "mappings": {
    "properties": {
      "ip_address": {
        "type": "ip"
      }
    }
  }
}
(2) 完成类型（Completion）
  • 类型名：completion。
  • 适用场景：
    • 用于实现自动补全功能。
  • 示例：
{
  "mappings": {
    "properties": {
      "suggest": {
        "type": "completion"
      }
    }
  }
}

1.2.6. 映射总结

字段类型适用场景

Numeric数值计算、范围查询（如年龄、价格）。

Boolean布尔值存储（如是否启用、是否完成）。

Range范围值存储（如时间段、价格区间）。
ObjectJSON 对象存储（如用户地址）。
Nested数组中的对象存储（如评论列表）。
Geo-point地理位置存储（如经纬度坐标）。
Geo-shape复杂地理形状存储（如多边形、线）。

2.3. 获取文档

GET /my_index3/_doc/1
{
  "_index" : "my_index3",
  "_type" : "_doc",
  "_id" : "1",
  "_version" : 2,
  "_seq_no" : 1,
  "_primary_term" : 1,
  "found" : true,
  "_source" : { }
}
  • "_index": "my_index3": 表示该文档属于名为my_index3的索引。
  • "_type": "_doc": Elasticsearch 7.x及以后版本中，文档类型已被弃用，默认使用_doc作为统一类型标识符。这个改变简化了Elasticsearch的API，并且更符合其作为文档存储的本质。
  • "_id": "1": 文档的唯一标识符是1，这可以是你在添加文档时指定的ID，或者是Elasticsearch自动生成的。
  • "_version": 2: 这个字段表示文档的版本号。每次文档被更新（包括删除），版本号都会增加。在这个例子中，文档已经被更新或修改了一次（初始版本为1，现在是第2版）。
  • "_seq_no": 1: 序列号，用于内部版本控制机制的一部分，以确保操作顺序的一致性。
  • "_primary_term": 1: 主分片任期编号，也是用于内部版本控制，保证在主分片和副本分片之间操作的一致性和正确性。
  • "found": true: 表明根据提供的_id，找到了匹配的文档。
  • "_source" : { }: 这部分通常包含文档的实际内容，也就是当你索引文档时所提供的数据。然而，在你的例子中，_source字段是一个空对象，这意味着虽然文档存在，但是它的内容为空或者没有通过请求返回。

3.1 查询

3.2 查询类型

查询类型描述
Match Query用于全文搜索，支持分词和模糊匹配。
Term Query用于精确匹配，不进行分词。
Range Query用于范围查询，如日期、数字等。
Bool Query用于组合多个查询条件，支持 must、should、must_not 等逻辑操作。
Wildcard Query用于通配符查询，支持 * 和 ?。
Prefix Query用于前缀匹配。
Fuzzy Query用于模糊匹配，支持拼写错误。
Nested Query用于嵌套对象的查询。
Geo Query用于地理位置查询。

3.3 查询结构

要点描述
Query DSLElasticsearch 使用 JSON 格式的 Query DSL 进行查询。
复合查询可以使用 bool 查询将多个查询条件组合在一起。
分页使用 from 和 size 参数进行分页。
排序使用 sort 参数对结果进行排序。
高亮使用 highlight 参数高亮显示匹配的文本。

3.4 性能优化

  • 索引设计: 合理设计索引结构，避免过多的嵌套和复杂的映射。
  • 分片和副本: 合理设置分片和副本数量，以提高查询性能和容错能力。
  • 缓存: 利用 Elasticsearch 的缓存机制，如 filter 查询的缓存。
  • 查询优化: 避免使用高开销的查询，如 wildcard 和 regexp 查询。

3.5 示例：

3.5.1 Match Query

用于全文搜索 full_text_content 字段中的内容。

{
  "query": {
    "match": {
      "full_text_content": "Elasticsearch 创建索引"
    }
  }
}

3.5.2 Term Query

用于精确匹配 user_id 字段。

{
  "query": {
    "term": {
      "user_id": "user_001"
    }
  }
}

3.5.3 Range Query

用于范围查询 age 和 price 字段。

{
  "query": {
    "range": {
      "age": {
        "gte": 25,
        "lte": 30
      }
    }
  }
}

3.5.4 Bool Query

用于范围查询 age 和 price 字段。

{
  "query": {
    "bool": {
      "must": [
        { "range": { "age": { "gt": 25 } } },
        { "term": { "is_enabled": true } }
      ]
    }
  }
}

3.5.5 Wildcard Query

使用通配符查询 user_id 字段。

{
  "query": {
    "wildcard": {
      "user_id": "user_*"
    }
  }
}

3.5.6 Prefix Query

查询 user_id 字段中以 user_ 开头的文档。

{
  "query": {
    "prefix": {
      "user_id": "user_"
    }
  }
}

3.5.7 Fuzzy Query

模糊查询 full_text_content 字段，允许拼写错误。

{
  "query": {
    "fuzzy": {
      "full_text_content": {
        "value": "Elasticserch",
        "fuzziness": "AUTO"
      }
    }
  }
}

3.5.8 Nested Query

查询 comments 嵌套对象中 upvotes（字段名字） 大于 5 的文档。

{
  "query": {
    "nested": {
      "path": "comments",
      "query": {
        "range": {
          "comments.upvotes": {
            "gt": 5
          }
        }
      }
    }
  }
}

3.5.9 Geo Query

查询 location 字段中距离指定坐标 10 公里范围内的文档。

{
  "query": {
    "geo_distance": {
      "distance": "10km",
      "location": {
        "lat": 40.7128,
        "lon": -74.0060
      }
    }
  }
}

3.5.11 Regexp Query

使用正则表达式查询 server_ip 字段。

{
  "query": {
    "regexp": {
      "server_ip": "192.168.*"
    }
  }
}

3.5.12 Suggest Query

使用 suggest_field 字段进行搜索建议。

{
  "suggest": {
    "my-suggestion": {
      "text": "Elasticsearch",
      "term": {
        "field": "suggest_field.input"
      }
    }
  }
}

3.5.13 Geo Shape Query

查询 area_covered 字段中与指定多边形相交的文档。

{
  "query": {
    "geo_shape": {
      "area_covered": {
        "shape": {
          "type": "polygon",
          "coordinates": [
            [
              [13.4101, 52.5179],
              [13.4101, 52.5189],
              [13.4111, 52.5189],
              [13.4111, 52.5179],
              [13.4101, 52.5179]
            ]
          ]
        },
        "relation": "intersects"
      }
    }
  }
}

3.5.14 Date Range Query

查询 created_at 字段在指定日期范围内的文档。

{
  "query": {
    "range": {
      "created_at": {
        "gte": "2025-03-01 00:00:00",
        "lte": "2025-03-31 23:59:59"
      }
    }
  }
}

3.5.15 Script Query

使用脚本查询 age 大于 25 的文档。

{
  "query": {
